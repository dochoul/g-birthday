# Gabia Birthday 개발 보고서

---

## 1. 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Gabia Birthday |
| 개발 목적 | 임직원 생일 기프티콘 지급 업무 지원 |
| 개발 형태 | 로컬 실행 데스크톱 앱 (Electron) |
| 배포 플랫폼 | macOS (Apple Silicon), Windows x64 |
| 버전 | 1.0.1 |
| 개발 완료일 | 2026-04-03 |

### 개발 배경

매달 생일인 직원에게 기프티콘을 지급하는 업무에서 아래와 같은 불편이 있었습니다.

- 인사팀 엑셀 명부를 수작업으로 확인하여 대상자 추출
- 고용형태(정규직/수습/인턴)나 휴직 여부를 별도로 구분해야 하는 번거로움
- 주민등록번호가 포함된 명부를 그대로 사용함에 따른 개인정보 노출 위험

이를 해소하기 위해 엑셀 명부를 기반으로 자동으로 대상자를 필터링하고, 개인정보를 안전하게 관리하는 전용 데스크톱 앱을 개발했습니다.

---

## 2. 주요 기능

### 2.1 월별 생일자 조회

- 1~12월 중 원하는 월을 선택하면 해당 월 생일자 목록을 표시
- 생일 오름차순 정렬
- 번호, 이름, 생일, 상태, 이메일, 소속, 사번, 고용형태 컬럼 표시
- 기프티콘 지급 비대상자(정규직-수습, 인턴, 휴직중) 행은 비활성화 처리하여 시각적으로 구분

### 2.2 엑셀 내보내기

- 선택한 월의 생일자 목록을 `.xlsx` 파일로 다운로드
- 소속 컬럼을 팀/유닛으로 분리하여 출력
- 파일명 형식: `{연도}년_{월}월_생일자.xlsx`
- 출력 컬럼: 번호, 이름, 팀, 유닛, 생일, 고용형태

### 2.3 직원 명부 업로드

- 인사팀 엑셀 파일(.xlsx)을 드래그&드롭 또는 클릭으로 업로드
- 업로드 시 필수 컬럼 유효성 검사
- 기존 파일은 자동 백업 (최대 10개 보관)
- 업로드 이력 관리: 일시, 파일명, 고용형태별(정규직/정규직-수습/인턴) 인원 수 기록

### 2.4 통계 페이지

- 전체/정규직/정규직-수습/인턴/휴직중 인원 요약 카드
- 월별 고용형태별 생일자 현황 스택 막대 차트 (1~12월)
- 정규직-수습, 인턴, 휴직중 직원 명단 및 생일 표시

### 2.5 개인정보 보호

- 업로드 즉시 주민등록번호 컬럼 삭제, 생일(YYYY-MM-DD)만 추출하여 저장
- 백업 파일에도 동일 처리 적용
- 앱 시작 시 기존 파일에 주민등록번호가 잔존하면 자동 마이그레이션
- API 응답, UI, 엑셀 내보내기 어디에도 주민등록번호 미포함

### 아키텍처 특징

- **npm workspaces 모노레포**: `client/`(프론트엔드)와 `server/`(Express 백엔드)를 단일 레포에서 관리
- **Electron + Express 내장 서버**: Electron 메인 프로세스가 Express 서버를 자식 프로세스로 실행하고, 프론트엔드 정적 파일을 서빙
- **로컬 전용**: 외부 서버 없이 사용자 PC에서 완전히 독립 실행

## 6. 데이터 처리

### 6.1 직원 명부 엑셀 형식

| 컬럼명 | 필수 | 설명 |
|--------|------|------|
| 이름(호칭) | O | 국문, 영문(국문), 국문(영문) 형식 모두 지원 |
| 사번 | O | 직원 사번 |
| 소속 | O | `팀>유닛` 형식 |
| 생일 | O* | YYYY-MM-DD |
| 주민등록번호 | O* | 생일 컬럼 없을 때 대체 (업로드 즉시 삭제) |
| 이메일 | O | 사내 이메일 |
| 상태 | O | `재직중` / `휴직중` |
| 고용형태 | O | `정규직` / `정규직-수습` / `인턴` |

### 6.2 개인정보 처리 흐름

```
엑셀 업로드
    │
    ├── 주민등록번호 컬럼 감지
    │       │
    │       └── 생일(YYYY-MM-DD) 추출 → 생일 컬럼으로 대체
    │               주민등록번호 컬럼 삭제
    │
    ├── 기존 파일 백업 (주민등록번호 제거 후 저장)
    │
    └── 새 파일 저장 (주민등록번호 없는 버전)
```

### 6.3 데이터 저장 위치

| OS | 경로 |
|----|------|
| macOS | `~/Library/Application Support/gabia-birthday/` |
| Windows | `%APPDATA%\gabia-birthday\` |

## 9. 구현 시 주요 고려사항

### 개인정보 보호

인사팀 엑셀에 주민등록번호가 포함되어 있어 유출 시 법적 책임이 발생할 수 있습니다. 이를 방지하기 위해 업로드 즉시 서버 메모리 상에서 생일만 추출하고 원본 컬럼을 제거한 뒤 저장하는 방식을 채택했습니다. 백업 파일과 앱 시작 시 마이그레이션까지 처리하여 어떤 경로로도 주민등록번호가 저장되지 않도록 했습니다.

### 데스크톱 앱 채택 이유

웹 서버로 배포할 경우 네트워크를 통해 개인정보가 전송되고 서버 관리가 필요합니다. 데스크톱 앱은 데이터가 담당자 PC에만 저장되고, 별도 서버 인프라 없이 파일 하나로 배포가 가능하여 선택했습니다.

### 이름 중복 처리

엑셀 명부에 `국문(영문)` 또는 `영문(국문)` 혼합 형식이 존재합니다. 동일인이 두 형식으로 두 행에 등록된 경우를 처리하기 위해 한글 이름을 추출하여 중복을 제거합니다.

### 이전 형식 하위 호환

기존 명부에 `생일` 컬럼 없이 `주민등록번호`만 있는 경우와, `생일` 컬럼이 추가된 신규 형식 모두 지원합니다.
